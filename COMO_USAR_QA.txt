EXPLICAÇÃO SIMPLES: 3 FORMAS DE USAR O SISTEMA DE PERGUNTAS


═══════════════════════════════════════════════════════════════════════════
FORMA 1: SCRIPT DE TESTE (MAIS FÁCIL!) ⭐ RECOMENDADO
═══════════════════════════════════════════════════════════════════════════

1. Certifique-se de que o Ollama está rodando:
   
   # Em uma janela do PowerShell:
   ollama serve
   
   # Deixe rodando!

2. Em OUTRA janela, execute o script de teste:
   
   python test_qa_system.py

3. O que acontece:
   
   • Carrega dados de exemplo automaticamente
   • Faz algumas perguntas de demonstração
   • Pergunta se você quer sessão interativa
   • Se você responder 's', abre um CHAT no console!

4. No chat, você digita suas perguntas:
   
   ┌──────────────────────────────────────────────────────────┐
   │ 💬 SESSÃO INTERATIVA DE PERGUNTAS E RESPOSTAS            │
   ├──────────────────────────────────────────────────────────┤
   │                                                          │
   │ Comandos especiais:                                      │
   │   • 'sair' ou 'exit' - Encerra                          │
   │   • 'melhorias' - Gera sugestões                        │
   │   • 'gargalos' - Identifica problemas                   │
   │   • 'historico' - Mostra perguntas anteriores           │
   │                                                          │
   │ 💬 Você: Qual veículo tem a maior distância?            │
   │                                                          │
   │ 🤖 Assistente:                                           │
   │ A Van Refrigerada 01 tem a maior distância              │
   │ com 42.5 km, seguida pela Caminhonete 03                │
   │ com 38.7 km...                                           │
   │                                                          │
   │ 💬 Você: Algum veículo está subutilizado?               │
   │                                                          │
   │ 🤖 Assistente:                                           │
   │ Sim, a Caminhonete 03 está com apenas 45.2%             │
   │ de capacidade utilizada...                               │
   │                                                          │
   │ 💬 Você: melhorias                                       │
   │                                                          │
   │ 🤖 Assistente:                                           │
   │ [Gera sugestões de melhoria automaticamente]            │
   │                                                          │
   │ 💬 Você: sair                                            │

   │ 👋 Encerrando sessão. Até logo!                         │
   └──────────────────────────────────────────────────────────┘

5. Ao sair, o log de perguntas é salvo em:
   outputs/reports/qa_log_demo.md


═══════════════════════════════════════════════════════════════════════════
FORMA 2: APÓS O main.py ⭐ NOVO!
═══════════════════════════════════════════════════════════════════════════

1. Execute o sistema completo:
   
   python main.py

2. Escolha parâmetros no menu Pygame

3. Aguarde otimização (pode demorar!)

4. Quando a janela Pygame abrir com os resultados:
   
   • Analise as rotas
   • Pressione Q para FECHAR o Pygame

5. Após fechar, o console perguntará:
   
   ┌──────────────────────────────────────────────────────────┐
   │ ============================================              │
   │ SIMULACAO CONCLUIDA!                                     │
   │ ============================================              │
   │                                                          │
   │ 💬 Deseja fazer perguntas sobre as rotas                │
   │    otimizadas? (s/n):                                    │
   └──────────────────────────────────────────────────────────┘

6. Digite 's' e pressione ENTER

7. O sistema indicará usar test_qa_system.py
   (porque os dados da simulação ainda não estão integrados)

NOTA: Por enquanto, após o main.py, ele recomenda usar o
      test_qa_system.py. Futuramente pode ser integrado
      diretamente!

FORMA 3: NO SEU PRÓPRIO CÓDIGO (Para desenvolvedores)
═══════════════════════════════════════════════════════════════════════════

Se você quiser fazer perguntas programaticamente:

from src.llm_integration import QASystem

# 1. Configurar
qa = QASystem(provider="ollama", model="llama2")

# 2. Carregar seus dados
qa.load_context(
    routes=[...],           # Suas rotas otimizadas
    vehicles=[...],         # Seus veículos
    delivery_points=[...],  # Seus pontos
    metrics={...}          # Suas métricas
)

# 3. Fazer perguntas
resposta = qa.ask("Qual veículo está mais carregado?")
print(resposta)

# 4. Ou iniciar chat interativo
from src.llm_integration import interactive_qa_session
interactive_qa_session(qa)

PROBLEMAS COMUNS
═══════════════════════════════════════════════════════════════════════════

PROBLEMA: "Failed to connect to Ollama"
SOLUÇÃO:
  • Ollama não está rodando
  • Abra outra janela: ollama serve
  • Deixe rodando!

PROBLEMA: Demora muito para responder
SOLUÇÃO:
  • Normal na primeira pergunta! (30-60 seg)
  • Próximas perguntas são mais rápidas (10-25 seg)
  • O modelo precisa carregar na memória

PROBLEMA: Não aparece nada, só fica esperando
SOLUÇÃO:
  • O LLM está processando! Aguarde!
  • Primeira vez sempre demora mais
  • Não interrompa (Ctrl+C cancela)

PROBLEMA: "No module named 'ollama'"
SOLUÇÃO:
  • pip install ollama
